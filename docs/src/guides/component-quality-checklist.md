# Component Quality Checklist

Findings from plan reviews (March 2026). Use this checklist when building or reviewing Astro/React components.

## Accessibility (A11y)

- [ ] **`<summary>` aria-label**: All collapsible `<details>/<summary>` components MUST provide a meaningful `aria-label` when no visible title text is present
  - CodeBlock: `"Code block"`, CLISnippet: `"Terminal output"`, DirTree: `"Directory tree"`, Timeline: `"Timeline: {range}"`
- [ ] **Fold controls**: `role="button"`, `aria-expanded="true|false"`, `aria-label="Collapse block"` on gutter fold icons
- [ ] **Hidden regions**: `aria-hidden="true"` on collapsed/hidden lines
- [ ] **Keyboard navigation**: All interactive components must be operable via keyboard (Tab, Enter, Space, Escape, Arrow keys)
- [ ] **`prefers-reduced-motion`**: Components with animation MUST honor this media query — disable transitions, show content instantly
- [ ] **Focus indicators**: Use existing `outline-ac focus-visible:outline-2 focus-visible:outline-dashed` pattern from `global.css`

## Brand Theming (ADR-0008)

- [ ] **Zero hardcoded colors**: All Chrome-tier UI colors use CSS variables (`text-fg`, `bg-mt`, `border-bd`, etc.)
- [ ] **Shiki dual themes**: Code highlighting uses `defaultColor: false` with `github-light`/`github-dark` producing `--shiki-light`/`--shiki-dark` CSS variables
- [ ] **Feedback colors**: Error states use `--error`, success states (e.g., "Copied!") use `--success`
- [ ] **Opacity via Tailwind**: `bg-mt/50`, `text-fg/40` (Tailwind v4 generates `color-mix()` — acceptable per ADR-0008)
- [ ] **No hardcoded palette**: Even in `typography.css` — diff highlight colors should use `--success`/`--error`, not `green-400`/`red-500`

## Consistency

- [ ] **Collapsible wrapper**: Use `.component-details` class to reset prose `<details>` overrides from `typography.css` (`inline-block`, `select-none`, `[&_p]:hidden`)
- [ ] **Content via string props**: Components parsing raw text (CodeBlock, CLISnippet, DirTree, Timeline) accept string/array props, NOT `<slot />`
- [ ] **PascalCase filenames**: `CodeBlock.astro`, `CLISnippet.astro`
- [ ] **`client:only="react"`**: Never `client:load` — SSR fails for Shiki WASM, canvas APIs, MutationObserver
- [ ] **Shared Shiki singleton**: All components use `src/utils/shiki.ts` with `??=` promise cache — never create per-component highlighters
- [ ] **Event delegation**: Prefer `document.addEventListener` with `.closest()` matching over per-element binding — avoids `astro:after-swap` re-binding issues

## Tailwind v4 Scoped Styles

**Tailwind v4 scoped `<style>` blocks in Astro components CANNOT access custom `@theme` tokens** (`border-bd`, `text-fg`, `bg-mt`, etc.). The `@theme inline` tokens defined in `global.css` are only available in the main CSS pipeline — not in component-scoped style blocks.

**What fails:**
```astro
<!-- BROKEN: border-bd is unknown in scoped <style> -->
<style>
  .my-component { @apply border border-bd; }
</style>
```

**What works:**
- Move component styles to `src/styles/components.css` (imported by `global.css`)
- Use `class:list` with Tailwind utilities directly in templates (always works)
- For component-specific classes that need `@apply` with custom tokens, define them in `components.css`

**Why**: Tailwind v4's Vite plugin processes each `<style>` block as an independent CSS module. The `@theme inline` block in `global.css` defines tokens that are only visible within the global pipeline. The `@reference` directive can import base Tailwind classes but NOT custom `@theme` definitions from other files.

Discovered 2026-03-25 during E2E testing — all 8 component `<style>` blocks had to be migrated to `src/styles/components.css`.

## Cross-Browser

- [ ] **`clip-path: inset()`**: Works on inline spans in Chrome 55+, Firefox 54+, Safari 13.1+. Use `@supports` guard for graceful degradation
- [ ] **`color-mix()`**: Chrome 111+, Firefox 113+, Safari 16.2+ (generated by Tailwind v4 opacity modifiers — accepted risk per ADR-0008)
- [ ] **`navigator.clipboard`**: Requires secure context (HTTPS or localhost). Provide `execCommand('copy')` fallback for non-secure dev environments
- [ ] **`details/summary`**: Universal browser support. Style with `list-style: none` and custom chevron
- [ ] **`IntersectionObserver`**: Universal, no polyfill needed

## Performance

- [ ] **Shiki singleton**: Module-level `??=` promise cache — `createHighlighter()` called once per build
- [ ] **OGCard build cache**: Module-level `Map<URL, OGData>` — same URL fetched once per build
- [ ] **OGCard HTML truncation**: `.slice(0, 51200)` before regex — prevents multi-MB page scans
- [ ] **OGCard fetch timeout**: `AbortController` with 5s timeout — prevents build hangs
- [ ] **Lazy-loaded assets**: Asciinema player (JS + CSS) loaded via `IntersectionObserver`. Shiki runtime (dev mode only) via dynamic import
- [ ] **DOM batching**: Read all `getBoundingClientRect` before writing any styles — avoids forced reflows in animation
- [ ] **Token map limits**: Soft warning at 200 lines, degrade to plain `<pre>` beyond 500

## Security

- [ ] **`dangerouslySetInnerHTML`**: Only from `codeToHtml()` output. Never from external fetches or user input
- [ ] **OGCard regex**: Not vulnerable to ReDoS with truncated input (<50KB)
- [ ] **Fetch of user-supplied URLs**: OGCard fetches author-specified URLs at build time — acceptable since MDX is trusted content
