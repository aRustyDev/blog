# Dictionary/Glossary + Backlinks System

## Overview

A knowledge layer that adds term definitions, bidirectional linking, and graph integration to the blog. Authors reference terms via `<TermLink>` components; the system resolves them at build time against a terms database, rendering interactive popups for known terms and plain text for undefined ones.

## Architecture

- **Data**: `terms.json` follows the `people.json` pattern (schema.org/DefinedTerm)
- **Components**: `TermPopup.astro` mirrors `PersonPopup.astro` (CSS-first hover, event delegation)
- **Build scripts**: `build-backlinks.ts` follows `build-graph.ts` (fs/path/yaml, JSON output)
- **Graph**: Terms become a third node type alongside posts and projects
- **Authoring**: `<TermLink term="id">text</TermLink>` is the MDX surface — resolves at build time

## Key Decisions

- **No runtime JS for term resolution** — all lookups happen at build time in Astro frontmatter (ADR-0010 compliant)
- **Backlinks index is static JSON** (`public/backlinks.json`) generated by build script, paralleling `public/graph.json`
- **TermLink is the authoring surface** — authors write `<TermLink term="id">text</TermLink>` in MDX; component handles exists/not-exists cases
- **Build pipeline order**: backlinks → graph → astro build (backlinks feeds graph)
