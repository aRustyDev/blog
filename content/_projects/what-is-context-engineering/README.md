# What is Context Engineering? A Taxonomy of AI Context Components

> Proposing "Context Engineering" as a discipline with a formal taxonomy of modular components that shape AI assistant behavior.

## Status

**Phase**: ideation
**Persona**: not set

## Overview

This project defines "Context Engineering" as a discipline distinct from (and broader than) prompt engineering. It proposes a taxonomy of six component types — skills, rules, hooks, memory, tools, and references — classified by function rather than format. The piece establishes design principles drawn from software engineering and grounds abstractions in concrete examples from the Claude Code plugin ecosystem.
