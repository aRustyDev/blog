> Sweet spots, overkill zones, and honest tradeoffs — a decision framework for Rust adoption.

## Status

**Phase**: ideation

## Overview

An honest assessment of where Rust shines (CLI, WebAssembly, embedded, networking), where it's overkill (CRUD apps, scripts, prototypes), and where it struggles (GUI, mobile, rapid iteration). Covers ecosystem maturity by domain, hiring/team considerations, compilation time costs, and concrete decision frameworks. Honesty-first approach: knowing when NOT to use Rust is as valuable as knowing when to.
