> From Pico clone to universal fallback — understanding the modeless editor that chose discoverability over power.

## Status

## Overview

A deep-dive on nano covering its origin (GNU replacement for Pico), the modeless philosophy, nano's feature set (syntax highlighting, auto-indentation, search/replace, multiple buffers), .nanorc configuration, nano vs modal editors (when simplicity wins), nano in system administration, and nano's role as the editor that doesn't require a tutorial. Accessibility-first approach: nano chose discoverability over power, and that's a valid design decision.
